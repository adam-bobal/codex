2. Architecture Patterns

Choosing the right orchestration pattern is critical. Pattern A (Client-Side) is easier to debug but moves more data over the network. Pattern B (Server-Chaining) keeps heavy data within the mesh but increases code complexity.

Pattern A: Client-Side

Host gets data from Node A, then sends it to Node B. Best for text/JSON.

Pattern B: Server-Chaining

Node A sends directly to Node B. Best for Images/Large Datasets.
